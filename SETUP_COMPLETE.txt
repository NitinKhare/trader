DEPLOYMENT WORKFLOWS
================================================================================

LOCAL DEVELOPMENT
  1. ./scripts/setup.sh (choose "1) Local machine")
  2. ./scripts/start_engine.sh
  3. Monitor: ./scripts/health_check.sh --verbose
  4. Stop: ./scripts/stop_engine.sh

SERVER PRODUCTION
  1. SSH to server: ssh user@server.com
  2. cd algoTradingAgent && ./scripts/setup.sh (choose "2) Server")
  3. Cron jobs auto-configured
  4. Monitor: ./scripts/health_check.sh
  5. View logs: tail -f logs/market_open_*.log

DOCKER CONTAINERIZATION
  1. ./scripts/setup.sh (choose "3) Docker")
  2. docker-compose up -d
  3. Monitor: docker-compose logs -f trading-engine
  4. Stop: docker-compose down

CLOUD DEPLOYMENT (AWS/GCP/Azure)
  1. ./scripts/cloud_deploy.sh
  2. Follow provider-specific guide in output
  3. Deploy using cloud provider's tools
  4. Monitor via cloud console

IMPORTANT NOTES
================================================================================

1. LIVE TRADING SAFETY
   • Always test with "paper" mode first
   • Live trading requires explicit confirmation
   • Monitor regularly, don't leave unattended
   • Set proper stop loss and daily loss limits

2. CONFIGURATION
   • Update config/config.json with your credentials
   • Get broker credentials from https://dhanhq.co/
   • Use strong passwords for database access
   • Never commit credentials to git

3. DATABASE
   • PostgreSQL should be installed and running
   • Database credentials must match config.json
   • Run migrations via setup script
   • Regular backups recommended

4. MONITORING
   • Monitor logs daily: tail -f logs/engine_*.log
   • Run health checks: ./scripts/health_check.sh
   • Set up alerts for errors
   • Keep dependencies updated

5. SECURITY
   • Use environment variables for secrets
   • Don't expose credentials in logs
   • Use cloud provider secret management
   • Enable audit logging
   • Regular security updates
